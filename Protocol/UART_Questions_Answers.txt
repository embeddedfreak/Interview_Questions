===============================
1. UART Fundamentals
===============================

1. What is UART?

UART is a hardware communication peripheral used for serial communication between devices.

It sends data bit-by-bit over TX and RX lines without using a clock signal.

Examples:
• MCU ↔ Sensor
• Linux Board ↔ Debug Console
• MCU ↔ Bluetooth Module

Simple interview answer:
"UART is a serial communication protocol/peripheral used to exchange data between devices using TX and RX lines."

--------------------------------------------------

2. What does UART stand for?

UART stands for:

Universal Asynchronous Receiver Transmitter

Universal → configurable for different devices/settings.
Asynchronous → no clock signal required.
Receiver → receives serial data.
Transmitter → transmits serial data.

Simple interview answer:
"UART stands for Universal Asynchronous Receiver Transmitter. It is hardware used for serial communication."

--------------------------------------------------

3. Why is UART called an asynchronous protocol?

UART is called asynchronous because sender and receiver do not share a clock line.

Both devices agree on parameters beforehand such as:
• Baud rate
• Data bits
• Parity
• Stop bits

Synchronization is achieved using start and stop bits.

Simple interview answer:
"UART is asynchronous because it does not use a separate clock signal. Both devices communicate using preconfigured timing parameters."

--------------------------------------------------

4. Difference between UART and USART?

UART:
• Supports asynchronous communication only.
• No clock line.

USART:
• Supports both asynchronous and synchronous communication.
• In synchronous mode, clock line is used.

Simple interview answer:
"UART supports only asynchronous communication, while USART supports both synchronous and asynchronous communication."

--------------------------------------------------

5. Difference between UART, SPI, and I2C?

UART:
• Asynchronous communication.
• Uses TX/RX lines.
• Point-to-point communication.
• No clock signal.

SPI:
• Synchronous communication.
• Uses MOSI, MISO, SCLK, CS.
• Very high speed.
• Master-slave architecture.

I2C:
• Synchronous communication.
• Uses SDA and SCL.
• Supports multiple devices using addressing.
• Two-wire bus.

Simple interview answer:
"UART is asynchronous and point-to-point. SPI is synchronous and high speed. I2C is synchronous and supports multiple addressed devices on the same bus."

--------------------------------------------------

6. Explain UART communication architecture.

UART communication has two sides:

Transmitter side:
CPU Parallel Data → UART TX Register → Shift Register → Serial Output

Receiver side:
Serial Input → Shift Register → RX Register → CPU Parallel Data

Main blocks:
• Baud Rate Generator
• TX Block
• RX Block
• Control Registers
• FIFO Buffer (optional)

Simple interview answer:
"UART converts parallel CPU data into serial data for transmission and converts received serial data back into parallel data."

--------------------------------------------------

7. What are TX and RX lines?

TX = Transmit line
RX = Receive line

TX sends data out.
RX receives incoming data.

Connection rule:
Device-1 TX → Device-2 RX
Device-1 RX → Device-2 TX

Simple interview answer:
"TX is used to transmit data and RX is used to receive data between devices."

--------------------------------------------------

8. What is full-duplex communication?

Full-duplex means both devices can transmit and receive simultaneously.

Example:
UART communication using separate TX and RX lines.

Simple interview answer:
"Full-duplex communication allows simultaneous data transmission and reception."

--------------------------------------------------

9. What is half-duplex communication?

Half-duplex means communication can happen in both directions, but not at the same time.

One side transmits while the other receives.

Example:
Walkie-talkie communication.

Simple interview answer:
"Half-duplex allows two-way communication, but only one device can transmit at a time."

--------------------------------------------------

10. What is simplex communication?

Simplex means communication happens in only one direction.

One device only sends.
The other device only receives.

Example:
Keyboard → Computer
TV Broadcast

Simple interview answer:
"Simplex communication supports only one-way data transfer."

===============================
2. UART Frame / Protocol Format
===============================

11. Explain UART frame format.

UART frame format defines how data is transmitted between devices.

A typical UART frame contains:

Start Bit → Data Bits → Optional Parity Bit → Stop Bit(s)

Example:

Start | Data | Parity | Stop

The frame helps the receiver identify the beginning, data content, error checking, and end of transmission.

Simple interview answer:
"UART frame format consists of a start bit, data bits, optional parity bit, and stop bit."

--------------------------------------------------

12. What is a start bit?

A start bit indicates the beginning of a UART transmission.

UART line normally stays HIGH (idle state).

When transmission begins, line changes from HIGH to LOW.

This LOW transition tells the receiver to start sampling incoming data.

Simple interview answer:
"A start bit is a LOW bit that indicates the beginning of UART data transmission."

--------------------------------------------------

13. What is a stop bit?

A stop bit indicates the end of a UART frame.

After transmitting data, the line returns to HIGH state.

Common configurations:
• 1 Stop Bit
• 1.5 Stop Bits
• 2 Stop Bits

It allows the receiver to detect frame completion.

Simple interview answer:
"A stop bit marks the end of UART transmission and returns the line to idle state."

--------------------------------------------------

14. Why is the start bit required?

UART is asynchronous and has no clock signal.

The receiver needs a way to know exactly when incoming data begins.

The start bit provides synchronization timing for sampling data bits.

Without a start bit, the receiver would not know where the data starts.

Simple interview answer:
"Start bit is required to synchronize the receiver and indicate the beginning of data transmission."

--------------------------------------------------

15. Why are stop bits required?

Stop bits indicate frame completion.

They provide timing margin for the receiver before the next frame starts.

They also help detect framing errors if expected stop bit timing is incorrect.

Simple interview answer:
"Stop bits are required to mark the end of a frame and allow receiver timing synchronization."

--------------------------------------------------

16. What is parity in UART?

Parity is a simple error detection mechanism used in UART communication.

An additional bit is added to check whether transmitted data contains bit errors.

Receiver calculates parity again and compares it with received parity.

If mismatch occurs, a parity error is reported.

Simple interview answer:
"Parity is an optional error-checking bit used to detect transmission errors."

--------------------------------------------------

17. Types of parity bits?

Common UART parity types:

• Even Parity
• Odd Parity
• No Parity
• Mark Parity (always 1)
• Space Parity (always 0)

Most commonly used:
Even parity, Odd parity, or No parity.

Simple interview answer:
"UART supports even parity, odd parity, no parity, and in some devices mark and space parity."

--------------------------------------------------

18. Difference between odd and even parity.

Even Parity:
Total number of 1s including parity bit must be EVEN.

Odd Parity:
Total number of 1s including parity bit must be ODD.

Example:

Data = 1011001

Number of 1s = 4

Even parity bit = 0
Odd parity bit = 1

Simple interview answer:
"In even parity, total number of 1s becomes even. In odd parity, total number of 1s becomes odd."

--------------------------------------------------

19. What is data length in UART?

Data length means the number of actual data bits transmitted in one UART frame.

Common values:
• 5 bits
• 6 bits
• 7 bits
• 8 bits
• 9 bits (some MCUs)

8-bit data length is most commonly used.

Simple interview answer:
"Data length is the number of payload data bits transmitted in a UART frame."

--------------------------------------------------

20. Explain 8N1 UART configuration.

8N1 is the most common UART configuration.

Meaning:

8 → 8 Data Bits
N → No Parity
1 → 1 Stop Bit

Frame structure:

Start Bit + 8 Data Bits + No Parity + 1 Stop Bit

Widely used in:
• Linux serial console
• Embedded debugging
• Bootloaders
• MCU communication

Simple interview answer:
"8N1 means 8 data bits, no parity, and 1 stop bit. It is the most common UART configuration."

===============================
3. Baud Rate / Timing
===============================

21. What is baud rate?

Baud rate is the speed of serial communication.

It represents how many signal changes or symbols are transmitted per second.

In UART, usually one symbol carries one bit, so baud rate is often equal to bit rate.

Example:
9600 baud → 9600 bits transmitted per second.

Simple interview answer:
"Baud rate defines the communication speed in UART, usually represented as bits per second."

--------------------------------------------------

22. Difference between baud rate and bit rate.

Baud Rate:
Number of signal changes per second.

Bit Rate:
Number of bits transmitted per second.

In UART, typically:

1 symbol = 1 bit

So baud rate and bit rate are usually the same.

In some communication systems, one symbol may represent multiple bits, so they can differ.

Simple interview answer:
"Baud rate is the number of signal changes per second, while bit rate is the number of bits transmitted per second. In UART, they are usually equal."

--------------------------------------------------

23. How is UART baud rate generated?

UART baud rate is generated using an internal clock and a baud rate divisor register.

The UART peripheral divides the input clock to create the required communication speed.

General formula:

Baud Rate = UART Clock / Divisor

Example:
Clock = 16 MHz
Configured divisor generates 115200 baud.

Simple interview answer:
"UART baud rate is generated by dividing the peripheral clock using baud rate divisor registers."

--------------------------------------------------

24. What happens if baud rates mismatch?

If transmitter and receiver baud rates do not match, communication errors occur.

Possible issues:
• Wrong characters received
• Framing errors
• Missing data
• Communication failure

Small mismatch may still work.

Large mismatch usually breaks communication.

Simple interview answer:
"If baud rates mismatch, receiver samples data at wrong timing, causing corrupted data or framing errors."

--------------------------------------------------

25. Allowed baud rate tolerance in UART?

UART allows only a small baud rate difference between transmitter and receiver.

Typical tolerance:
Around ±2% to ±5%.

The exact limit depends on:
• UART hardware
• Clock accuracy
• Frame size
• Oversampling method

Simple interview answer:
"UART supports only small baud rate errors, typically within a few percent, otherwise communication may fail."

--------------------------------------------------

26. Common UART baud rates.

Common UART baud rates are:

• 1200
• 2400
• 4800
• 9600
• 19200
• 38400
• 57600
• 115200
• 230400
• 460800
• 921600

Most commonly used:
9600 and 115200.

Simple interview answer:
"Common UART baud rates include 9600, 19200, 57600, and 115200 baud."

--------------------------------------------------

27. Explain oversampling in UART.

Oversampling means the receiver samples incoming data multiple times per bit period.

Instead of reading once, UART may sample 8 or 16 times.

This improves:
• Timing accuracy
• Noise immunity
• Reliable bit detection

Simple interview answer:
"Oversampling means sampling each bit multiple times to improve UART timing accuracy and reliability."

--------------------------------------------------

28. Why do UART peripherals use 16x oversampling?

16x oversampling improves reliable data reception.

Benefits:
• Better start bit detection
• Improved noise filtering
• More accurate bit timing
• Better tolerance to clock mismatch

The receiver samples the signal many times and determines the correct bit value.

Simple interview answer:
"UART uses 16x oversampling to improve synchronization, timing accuracy, and noise tolerance."

--------------------------------------------------

29. How do you calculate UART divisor values?

UART divisor is calculated using UART clock frequency and desired baud rate.

Generic formula:

Divisor = UART Clock / Desired Baud Rate

Some controllers include additional factors like 16x oversampling.

Actual formula depends on hardware.

Example:

UART Clock = 16 MHz
Desired Baud = 115200

Divisor is calculated and programmed into UART registers.

Simple interview answer:
"UART divisor values are calculated using the UART clock frequency and required baud rate."

--------------------------------------------------

30. How do clock sources affect UART timing?

UART timing depends on the clock source driving the peripheral.

Clock sources may be:
• Crystal oscillator
• PLL clock
• Internal RC oscillator

If clock accuracy is poor:
• Baud rate error increases
• Communication reliability decreases

Stable clocks improve UART accuracy.

Simple interview answer:
"UART timing depends on the peripheral clock source. Inaccurate clocks can cause baud rate errors and communication failures."

4. UART Hardware Concepts
===============================

31. Explain UART transmitter block diagram.

UART transmitter converts CPU parallel data into serial data.

Typical transmitter blocks:

CPU/Data Register → TX FIFO → Shift Register → TX Pin

Working:
• CPU writes data into TX register or FIFO.
• Data moves into shift register.
• Shift register sends data serially bit-by-bit.
• Data appears on TX line.

Simple interview answer:
"UART transmitter takes parallel data from CPU, converts it into serial format using a shift register, and transmits it through the TX pin."

--------------------------------------------------

32. Explain UART receiver block diagram.

UART receiver converts serial data into parallel data.

Typical receiver blocks:

RX Pin → Shift Register → RX FIFO/Register → CPU

Working:
• Serial data arrives on RX pin.
• Receiver samples incoming bits.
• Shift register reconstructs data.
• Data is stored in RX FIFO/register.
• CPU reads received data.

Simple interview answer:
"UART receiver samples serial input, converts it into parallel data using a shift register, and stores it for CPU access."

--------------------------------------------------

33. What are UART shift registers?

Shift registers are hardware registers used to send and receive serial data.

TX Shift Register:
Converts parallel data into serial bits.

RX Shift Register:
Collects serial bits and converts them into parallel data.

They are core blocks inside UART hardware.

Simple interview answer:
"UART shift registers are hardware blocks that convert parallel data to serial data and serial data back to parallel data."

--------------------------------------------------

34. What is UART FIFO?

FIFO stands for First In First Out.

UART FIFO is a hardware buffer used to temporarily store transmit or receive data.

Data enters and exits in the same order.

Purpose:
• Reduce CPU overhead
• Prevent data loss
• Support high-speed communication

Simple interview answer:
"UART FIFO is a hardware buffer that stores UART data in first-in-first-out order."

--------------------------------------------------

35. Difference between shift register and FIFO.

Shift Register:
• Performs serial ↔ parallel conversion.
• Handles bit-level transfer.
• Usually holds one frame at a time.

FIFO:
• Temporary storage buffer.
• Handles byte/packet buffering.
• Can hold multiple entries.

Simple interview answer:
"Shift register converts serial and parallel data, while FIFO is used to buffer multiple data bytes."

--------------------------------------------------

36. What is TX FIFO?

TX FIFO is the UART transmit buffer.

CPU writes data into TX FIFO.

UART hardware automatically pulls data from FIFO and sends it through TX line.

Benefit:
CPU does not need to feed every byte immediately.

Simple interview answer:
"TX FIFO is a transmit buffer that stores outgoing UART data before transmission."

--------------------------------------------------

37. What is RX FIFO?

RX FIFO is the UART receive buffer.

Incoming UART data is stored in RX FIFO.

CPU later reads buffered data.

Benefit:
Helps avoid receive data loss during heavy CPU load.

Simple interview answer:
"RX FIFO is a receive buffer that stores incoming UART data until the CPU reads it."

--------------------------------------------------

38. Advantages of FIFO buffering.

FIFO buffering provides multiple advantages:

• Reduces CPU interrupts
• Improves throughput
• Prevents data loss
• Supports high-speed communication
• Allows temporary buffering during CPU busy periods

Simple interview answer:
"FIFO buffering improves UART performance by reducing CPU overhead and preventing data loss."

--------------------------------------------------

39. What is FIFO trigger level?

FIFO trigger level defines when UART generates an interrupt or event.

Example:
RX FIFO trigger = 8 bytes

Interrupt occurs after 8 bytes are received.

Common trigger levels:
• 1 byte
• 4 bytes
• 8 bytes
• 16 bytes

Used to optimize interrupt frequency.

Simple interview answer:
"FIFO trigger level is the configured buffer threshold that decides when UART generates an interrupt."

--------------------------------------------------

40. Why are UART status registers important?

UART status registers provide information about UART state and errors.

Typical status flags:

• TX Empty
• RX Ready
• TX Complete
• Parity Error
• Framing Error
• Overrun Error

Drivers use these registers to control communication and debug issues.

Simple interview answer:
"UART status registers are important because they report UART state, data availability, and communication errors."

===============================
5. UART Modes / Communication Methods
===============================

41. Polling vs interrupt UART communication.

Polling:
CPU continuously checks UART status registers to see whether data is ready.

Interrupt:
UART hardware notifies CPU automatically when an event occurs.

Polling is simpler but wastes CPU cycles.
Interrupt method is more CPU efficient.

Simple interview answer:
"In polling, CPU repeatedly checks UART status. In interrupt mode, UART notifies CPU only when an event occurs."

--------------------------------------------------

42. Polling UART advantages/disadvantages.

Advantages:
• Simple implementation
• Easy debugging
• Suitable for simple systems

Disadvantages:
• High CPU usage
• Poor efficiency
• Not suitable for high-speed communication

Used mainly in:
• Small firmware
• Simple boot code
• Low-data-rate systems

Simple interview answer:
"Polling UART is simple to implement but wastes CPU time because the processor continuously checks UART status."

--------------------------------------------------

43. Interrupt UART advantages/disadvantages.

Advantages:
• Better CPU utilization
• Faster response to UART events
• Suitable for multitasking systems

Disadvantages:
• More complex implementation
• ISR latency must be managed
• Debugging can be harder

Commonly used in:
• RTOS systems
• Linux drivers
• Embedded applications

Simple interview answer:
"Interrupt UART improves CPU efficiency because hardware notifies the CPU only when data arrives or transmission completes."

--------------------------------------------------

44. DMA UART advantages/disadvantages.

Advantages:
• Very low CPU overhead
• High throughput
• Efficient large data transfer

Disadvantages:
• More complex configuration
• DMA resource management required
• Debugging complexity increases

Used for:
• High-speed logging
• Large packet transfer
• Continuous data streaming

Simple interview answer:
"DMA UART transfers data directly between memory and UART without heavy CPU involvement."

--------------------------------------------------

45. When should UART DMA be used?

UART DMA should be used when:

• Data rate is high
• Large transfers are required
• CPU load must be reduced
• Continuous streaming exists

Examples:
• Audio streaming
• GPS data logging
• High-speed debug logging
• Linux serial high-throughput transfers

Simple interview answer:
"UART DMA is used for high-speed or large data transfers where CPU overhead needs to be minimized."

--------------------------------------------------

46. Explain UART blocking communication.

In blocking communication, the CPU waits until UART operation finishes.

Example:
CPU calls UART transmit function and cannot continue until transmission completes.

Simple but CPU remains occupied.

Simple interview answer:
"Blocking UART communication makes the CPU wait until send or receive operation completes."

--------------------------------------------------

47. Explain UART non-blocking communication.

In non-blocking communication, CPU starts UART operation and continues executing other tasks.

Completion is handled later using:
• Interrupts
• DMA
• Callbacks

Improves multitasking performance.

Simple interview answer:
"Non-blocking UART allows the CPU to continue running while UART communication happens in the background."

--------------------------------------------------

48. How does asynchronous UART API work?

Asynchronous UART API starts communication and returns immediately.

Actual transfer happens independently.

Completion is usually notified through:
• Interrupts
• Events
• Callbacks

Example:
Application starts transmit request and later receives completion notification.

Simple interview answer:
"Asynchronous UART APIs start communication without waiting for completion and notify the application later."

--------------------------------------------------

49. Explain UART event-driven communication.

In event-driven communication, UART actions are triggered by events.

Common events:
• Data received
• Transmission complete
• Timeout
• Error detection

Application reacts only when events occur.

Widely used in:
• RTOS
• Embedded Linux
• Driver frameworks

Simple interview answer:
"Event-driven UART communication processes data only when specific UART events occur."

--------------------------------------------------

50. How do callback-based UART drivers work?

Callback-based drivers use user-defined functions that execute when UART events occur.

Flow:

Start UART operation → Event occurs → Driver calls callback function.

Common callbacks:
• RX complete callback
• TX complete callback
• Error callback

Reduces polling and improves asynchronous handling.

Simple interview answer:
"Callback-based UART drivers notify the application by executing registered functions when UART events occur."

• Head pointer
• Tail pointer

Read:

Wrap-around occurs at buffer end.

Used commonly with UART interrupts.

Simple interview answer:

Parity error occurs when the received parity value does not match the calculated parity.

It indicates possible data corruption during transmission.

Causes:
• Noise
• Signal disturbance
• Communication mismatch

Simple interview answer:
"Parity error occurs when transmitted and received parity values do not match, indicating possible data corruption."

Framing error occurs when UART does not detect the expected stop bit.

Usually, the receiver expects a HIGH stop bit but receives invalid timing or logic level.

Common causes:
• Baud rate mismatch
• Noise

UART TTL:

• Single-ended signaling.
• Short-distance communication.
• Limited noise immunity.
• Point-to-point connection.

RS485:

• Differential signaling.
• Longer communication distance.
• Better noise immunity.
• Supports multi-device communication.

Simple interview answer:
"UART TTL is short-range single-ended communication, while RS485 uses differential signaling for long-distance and noisy environments."

RS232 uses positive and negative voltage levels.

Logic 1 (MARK):
Usually −3V to −15V.

Logic 0 (SPACE):
Usually +3V to +15V.

This is different from TTL UART logic.

Simple interview answer:

84. Explain RS485 differential signaling.

RS485 uses two signal lines:

A and B.

Data is determined by voltage difference between the two lines.

Benefits:

• Better noise immunity
• Long-distance communication
• Reliable industrial communication

Because common noise affects both lines equally, the receiver can reject noise.

Simple interview answer:
"RS485 uses differential signaling where data is transmitted using voltage difference between two wires, improving noise immunity."

RS485 is preferred when:

Level shifters convert signals safely between voltage domains.
